# 開発仕様書

## 概要
画像差分ツール (Visual Diff Tool) のGUI仕様および構成。

## GUIの仕様
- **テーマ**: 全体を通してダークモードを採用。
- **レイアウト**:
  - `header`: タイトルと対応フォーマットの説明を配置。
  - `toolbar`: 画像選択（A・B）と比較実行ボタン、カテゴリ・表示切り替え、ズームコントロールをまとめて配置。これにより、画像の選択後すぐに比較を実行しやすい設計としている。
  - `viewer`: 左右に並べて画像を表示するエリア。左側に基準画像（A）、右側に比較結果（B）を表示する。
  - `diff memo`: 比較後に `差分メモ` ボタンから別タブで開く。画像Aと補正済み画像Bをスライダーで比較し、画像差分とは独立したテキストメモを配置できる。
- **メモ付きコピー**:
  - 差分メモ画面の右クリックメニューから、画像Aまたは画像Bにメモを描画したPNGをクリップボードへ保存する。
  - メモはフロントエンド上の注釈であり、Backend APIの差分計算結果には含めない。
  - 別タブ用の比較画像ペイロードは、通常 IndexedDB に保存し、利用できない場合のみ localStorage にフォールバックする。

## 実装構成
- **Frontend**: React (Vite)
  - `main.jsx`: UIのメインロジック。
  - `styles.css`: ダークモード対応のスタイリング。
- **Backend**: Python (FastAPI, OpenCV)
  - **ポート**: `8078` (ローカルサーバー)
  - `alignment.py`: SIFT / AKAZE / ORB による段階的な特徴量マッチングと ECC 補正。
  - `diffing.py`: YIQベースのピクセル差分、細線を維持するマスク生成、オーバーレイ、矩形抽出。
  - `raster_cache.py`: アップロード内容のハッシュをキーにしたラスター化キャッシュ。
  - `result_cache.py`: しきい値再計算用の補正済み画像キャッシュ。

## API仕様メモ
- `POST /api/diff`: ファイル変換、ページ選択、位置合わせ、初回差分生成を行う。レスポンスには補正済み画像と `result_id` を含める。
- `POST /api/rediff`: `result_id` と新しいしきい値から差分オーバーレイとマスクだけを再生成する。キャッシュ期限切れ時は `image_a` と `image_b_aligned` を直接送るフォールバックも受け付け、成功時は新しい `result_id` を返す。

## 注意事項
- コンポーネントおよびCSSは使いやすさを重視し、ボタンなどのアクション要素は操作順序に従って配置する。
- 各ソースコードの冒頭には日本語で仕様のコメントを記述する。
