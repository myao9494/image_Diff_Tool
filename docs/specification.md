# 詳細仕様書

## 0. 初期版の前提

- 比較対象の主語は常に「ファイル全体」とする。
- マルチページファイルでは、読み込み後に左右それぞれのページをユーザーが手動選択して比較する。
- 位置合わせカテゴリは `汎用`、`図面`、`グラフ`、`書類` の4種を提供し、デフォルト値は `汎用` とする。
- 初期版の差分ロジックはピクセル差分を採用する。
- 差分表示の色分けは「追加 = 青」「削除 = 赤」を基本とする。
- 位置合わせに失敗した場合は処理を中断せず、未補正のまま差分表示を継続する。
- 初期版では VSCode 拡張機能の実装はスコープ外とし、Web アプリと Backend API を優先する。
- API は今後の拡張を見据えて、変換、解析、差分の責務を分離した構成を維持する。

## 1. 画像処理パイプライン

比較プロセスは以下の順序で実行される。

1. **Input**: 2つのファイル (A, B) を受け取る（PNG, SVG, Excalidraw JSON, Obsidian Excalidraw Markdown, マルチTIFF, マルチPDF）。
2. **Pre-process**:
    - **マルチページ分解**:
        - PDFの場合: `PyMuPDF (fitz)` 等を用いて、各ページを高解像度（例: 300 DPI）のPNG画像として展開・一時保存。
        - マルチTIFFの場合: `Pillow (PIL.Image)` の `seek()` を用いて、全フレームを個別のPNG画像として抽出。
    - **単一画像変換**: SVG, Excalidraw JSON, Obsidian Excalidraw Markdown をPNGにラスタライズする。
    - **Excalidraw変換の初期方針**:
        - 通常の `.excalidraw` JSON を直接入力として受け付ける。
        - Obsidian の Excalidraw `.md` 形式も対象に含める。
        - 初期版では埋め込み画像アセットは対象外とし、図形とテキストをラスタライズした結果を比較対象とする。
    - **アルファチャンネル処理**: 透過背景を「白」で塗りつぶし、図面の視認性を確保。
3. **Feature Matching**:
    - UIで選択されたカテゴリ（`汎用`、`図面`、`グラフ`、`書類`）に応じて前処理または特徴量抽出パラメータを切り替える。
    - OpenCVの `ORB` または `SIFT` 検出器を使用してキーポイントを抽出する。
    - `findHomography` (RANSAC) で変換行列を計算し、回転・歪みを補正。
4. **Warping**: 画像Bを画像Aの座標系に射影変換。
5. **Diff Generation**:
    - 初期版ではピクセル差分を用いる。
    - 二画面表示用に、変更があったピクセルを強調する。
    - 色分けは「削除箇所は赤」「追加箇所は青」を基本とする。
6. **Fallback**:
    - 十分な特徴点が得られない、または変換行列を安定して推定できない場合は、未補正のまま差分表示を行う。
    - UI上では「位置合わせ失敗、未補正で表示中」のような警告を明示する。

## 2. API エンドポイント (案)

- `GET /api/health`: Backend の稼働確認。
- `POST /api/analyze`: アップロードされたファイルの基本メタデータ（総ページ数、フォーマットなど）を取得。
- `POST /api/diff`: 2つのファイル（および指定されたそれぞれのページ番号 `page_a`, `page_b`）を受け取り、補正後の画像と差分データを返す。
- `POST /api/convert`: 指定された特定ページまたは単一ファイルをPNGにラスタライズする。
- `GET /`: `frontend/dist` に含まれる Web UI を配信する。

### 初期版 API 方針

- 比較操作はファイル単位で開始し、マルチページファイルではページ選択結果を `page_a`、`page_b` として指定する。
- 初期版の `POST /diff` は、少なくとも以下の情報を返せるようにする。
    - 補正後画像B
    - 差分オーバーレイ画像
    - 変換行列
    - 差分マスク
    - 差分矩形一覧
- 将来の拡張を見据え、レスポンス形式は追加フィールドを後方互換で増やしやすい構造とする。
- Windows オフライン利用を考慮し、`frontend/dist` はリポジトリに含め、Backend から静的配信する。

## 3. UI/UX デザイン

- **File-first Flow**:
    - まずファイルAとファイルBを読み込む。
    - マルチページファイルの場合、その配下で左右それぞれのページを手動選択して比較する。
- **Category Selector**:
    - `汎用`、`図面`、`グラフ`、`書類` の4種類を選択可能にする。
    - デフォルト値は `汎用` とする。
- **Side-by-Side Diff**: 左にファイルA、右に補正済みファイルBを表示する。
- **Page Navigator**:
    - マルチページファイルの場合、画面下部またはサイドバーに「ページ一覧（またはサムネイル）」を表示。
    - 左右それぞれで比較したいページを個別に選択可能にする（例: Aファイルの1ページ目 vs Bファイルの1ページ目）。
- **Color Coding**:
    - 差分箇所を透過度を持たせた赤/青でオーバーレイ表示する。
    - 基本ルールは「追加 = 青」「削除 = 赤」とする。
- **Synchronized Scrolling**: 左右の画像のスクロール・ズームを同期。
- **Alignment Failure Notice**: 位置合わせに失敗した場合は、未補正表示であることを明示する警告を表示する。

### 参考 UI

- VSCode 拡張 `Image diff viewer` (作者: RayWiis) の操作感を参考にする。
- 特に、左右比較、ズーム、パン、同期スクロール、差分の見やすい切り替えを参考対象とする。

## 4. 今後の検討事項

- [ ] 図面特有の巨大画像、特に A0 ロール相当の長尺画像を最大 25 枚程度扱う場合の処理パフォーマンスとメモリ管理。
- [ ] 特徴量が少ない画像（真っ白な背景に小さな点など）でのマッチング精度。
- [ ] Excalidrawのフォントレンダリングの再現性。
- [ ] ピクセル差分以外の差分アルゴリズムへの拡張余地。
- [ ] カテゴリごとの最適な前処理、特徴量抽出、閾値パラメータの調整。

## 5. テスト用画像サンプル仕様

各カテゴリにおける15個のペア（合計30枚）について、画像A（基準）と画像B（一部変更＆位置ズレあり）の具体的な描画・変更仕様は以下の通りとする。

### 1) 機械設計図面 (Mechanical Drawings)
- **gear (歯車)**:
    - **A**: 基準歯車図面。中心円、ピッチ円、歯（24枚）、寸法線（`D = 120.0`）。
    - **B**: 歯車の右上の歯が1つ欠損。寸法値が `D = 125.0` に変更。反時計回りに3度回転。
- **flange (フランジ継手)**:
    - **A**: ボルト穴が4つの円形フランジ。
    - **B**: ボルト穴が6つに増加。右に8px、下に5px平行移動。
- **shaft (段付きシャフト)**:
    - **A**: 段付きシャフトの側面断面図。ハッチング、キー溝、全長の寸法線。
    - **B**: 段の長さが延長。キー溝の幅が2px縮小。全体が3%縮小。
- **bracket (L字ブラケット)**:
    - **A**: L字ブラケットの3面図（正面・平面・側面）。
    - **B**: 補強用リブ（斜め板）が正面図と側面図に追加。
- **bolt (ボルト・ナット)**:
    - **A**: 六角ボルトの頭部およびネジ部。ねじ山の長さ。
    - **B**: ねじ山部分のピッチが細かくなり、ボルト全長が10px延長。時計回りに1.5度傾斜。
- **pcb (配線基板)**:
    - **A**: 2層PCBの簡易的な配線パターン（導体配線、ハンダパッド、ビア）。
    - **B**: 一部の配線ルートが別ルートに変更、ビアが1つ削除。反時計回りに4度回転。

### 2) 解析データ (Graphs & Analysis)
- **stress_strain (応力ひずみ曲線)**:
    - **A**: 材料試験の応力-ひずみ曲線。降伏点、弾性変形領域、最大引張強さのプロットとラベル。
    - **B**: 最大引張強さのピークが上方にシフト。降伏点ラベルの位置が若干ズレ。右に15px平行移動。
- **thermal_heatmap (熱解析カラーマップ)**:
    - **A**: 中央部が高温（赤）で周辺が低温（青）の矩形グラデーションマップ。
    - **B**: 高温領域（赤）が右上方向に偏心。左に5px、上に5px移動。
- **fft_spectrum (振動周波数スペクトル)**:
    - **A**: 3つの鋭いピークを持つFFT解析スペクトル図。グリッド線、軸ラベル。
    - **B**: 3つ目のピークが消失、新しく高周波帯に小さなノイズピークが出現。全体が5%拡大。
- **bathtub_curve (バスタブ曲線)**:
    - **A**: 初期故障、偶発故障、摩耗故障の3段階を示す信頼性バスタブ曲線。
    - **B**: 摩耗故障の立ち上がり開始点が左にシフト。凡例テキストの変更。

### 3) 書類・ワークフロー (Documents & Workflows)
- **bom (部品表)**:
    - **A**: 5行の部品リスト（部品番号、部品名、数量、材質）。グリッド枠線。
    - **B**: 3行目の数量が `2` から `5` に変更。材質が `SUS304` から `S45C` に変更。
- **ecn (設計変更通知書)**:
    - **A**: 設計変更通知書の書式、日付、変更概要テキスト、右下に「承認」の丸い赤インク印影。
    - **B**: 変更概要テキストが2行追加。右下の「承認」印影が消失（未承認状態）。時計回りに5度回転。
- **workflow_flowchart (承認フローチャート)**:
    - **A**: 「設計」→「検図」→「承認」→「出図」の直線的フローチャート。
    - **B**: 「検図」から「修正」へ戻るフィードバックの分岐と、矢印が追加。
- **factory_layout (工場レイアウト)**:
    - **A**: 工場フロア内のCNC旋盤、マシニングセンタ、作業台の配置図（平面配置）。
    - **B**: CNC旋盤の位置が20px下に移動、空きスペースに「測定ベンチ」が追加。
- **inspection_certificate (検査成績書)**:
    - **A**: 外観、寸法、強度の検査結果表。すべての判定欄に「OK」が印字。
    - **B**: 強度の実測値が基準値外となり、判定欄が「NG」に変更されて赤文字強調。
